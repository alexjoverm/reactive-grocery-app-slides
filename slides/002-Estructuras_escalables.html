Estructuras escalables

Segunda práctica: tratar el State como si fuera una BBDD, en vez de como el UI lo desee. Para ello:

  • Normalizar los datos y mantenerlos en 1 nivel, sin anidar
  • Sólo utilizar arrays para ids, mediante una estructura { ids: [], entities: {} } (para States de tipo "lista")

Esto favorece el mantenimiento, mejora la escalabilidad y el rendimiento, y evita duplicados de datos

Tip: Normalizr es de gran ayuda, aunque no es necesario.
